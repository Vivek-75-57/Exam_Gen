## 📌 Bookmarks System

### What Are Bookmarks?

Bookmarks allow you to save important questions, answers, and notes for future reference.

### Features

- **Save Questions:** Mark questions during or after exams
- **Add Notes:** Include personal notes with bookmarks
- **Organize:** View all saved questions in one place
- **Export:** Download bookmarks for sharing

### How to Use Bookmarks

#### During Exam
```
1. See a tricky question?
2. Click the bookmark icon (📌)
3. It's saved automatically
4. Continue with exam
```

#### After Exam
```
1. View results screen
2. Click bookmark on any question
3. Optionally add notes
4. Question saved to your collection
```

#### Viewing Bookmarks
```
1. Go to "My Bookmarks" section
2. See all saved questions
3. Click to review details
4. Click to delete bookmark
```

### API Reference - Bookmarks

```javascript
// Get all bookmarks
GET /api/bookmarks

// Create bookmark
POST /api/bookmarks
{
  "questionId": "q123",
  "questionText": "What is JavaScript?",
  "savedAnswer": "A programming language",
  "notes": "Important concept"
}

// Delete bookmark
DELETE /api/bookmarks/:id
```

## 📖 AI-Generated Explanations

### What Are Explanations?

Detailed breakdowns of correct answers, generated by the Groq AI.

### What Explanations Include

| Component | Description |
|-----------|-------------|
| **Answer** | The correct answer |
| **Why** | Explanation of why it's correct |
| **Concepts** | Related concepts to learn |
| **Examples** | Real-world examples |
| **Tips** | Memory aids or tricks |

### Example Explanation

```
Question: What is event delegation in JavaScript?

Answer: Event delegation is a technique where you attach a 
single event listener to a parent element instead of adding 
listeners to individual child elements.

Why: This is efficient because:
1. Reduces memory usage
2. Works for dynamically added elements
3. Simplifies code maintenance

Related Concepts:
- Event bubbling
- Event capturing
- DOM event flow

Example:
// Instead of:
document.getElementById('item1').addEventListener(...)
document.getElementById('item2').addEventListener(...)

// Do this:
document.getElementById('list').addEventListener('click', handler)

Tips: Remember "delegate" = pass responsibility to parent
```

### Accessing Explanations

1. **After Exam:** Click question to see explanation
2. **From Results:** Click "View Explanation" button
3. **From Bookmarks:** Open bookmarked question

## 🚀 Vector Store & Caching

### What Is It?

Smart caching system that stores questions for faster retrieval.

### Benefits

| Benefit | Explanation |
|---------|-------------|
| **Speed** | Retrieve questions instantly |
| **Search** | Find similar questions |
| **Efficiency** | Reduce API calls |
| **Learning** | Better question recommendations |

### How It Works

```
Question Generated → Convert to Vector → Store in Cache
    ↓
Search Query → Find Similar Vectors → Return Results
```

### Enabling/Disabling

In `backend/.env`:
```env
ENABLE_FAISS_CACHING=true   # Enable caching
ENABLE_FAISS_CACHING=false  # Disable caching
```

## 🔄 Advanced API Usage

### Generate Exam with Custom Options

```bash
curl -X POST http://localhost:3001/api/exams \
  -H "Content-Type: application/json" \
  -d '{
    "subject": "JavaScript",
    "count": 10,
    "difficulty": "medium",
    "type": "multiple-choice"
  }'
```

### Submit Exam

```bash
curl -X POST http://localhost:3001/api/exams/123/submit \
  -H "Content-Type: application/json" \
  -d '{
    "answers": [
      {"questionId": "q1", "selectedOption": "A"},
      {"questionId": "q2", "selectedOption": "B"}
    ]
  }'
```

### Get Results

```bash
curl http://localhost:3001/api/exams/123/results
```

## 🐛 Troubleshooting Advanced Features

### Issue: Bookmarks not saving
**Solution:** Check browser localStorage is enabled

### Issue: Explanations not displaying
**Solution:** Verify Groq API is working (check health endpoint)

### Issue: Slow performance
**Solution:** Enable ENABLE_FAISS_CACHING in .env

### Issue: Reports not generating
**Solution:** Ensure all answers were submitted
